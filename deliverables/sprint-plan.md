# Sprint Plan: "Алиса покажи пизду"

> Юмористический вирусный одностраничник с публичным чатом.
> 2 недели, 1 разработчик, Next.js + Node.js, VPS.

---

## Specification Summary

**Original Requirements** (из idea.txt):
- Веб-страница с чатом, стилизованным под Яндекс Алису
- Кнопка "Спросить Алису" -- отправляет фиксированное сообщение "Алиса покажи пизду"
- Нейросеть отвечает (скорее всего отказывает смешно)
- После ответа можно написать свой текст -- стоит 1000р, 1 попытка на пользователя
- ВСЯ история переписки публичная, real-time

**Technical Stack**: Next.js, Node.js (API), PostgreSQL (или SQLite для MVP), WebSocket/SSE, Tailwind CSS
**Target Timeline**: 2 недели до рабочего MVP
**Deploy**: VPS (Docker или PM2)

#### [ ] 1.3 API: эндпоинт POST /api/ask (MUST, 3ч, день 2)

**Описание**: Эндпоинт принимает запрос, отправляет фиксированный текст "Алиса покажи пизду" нейросети, сохраняет пару (вопрос + ответ) в БД, возвращает результат.

**Acceptance Criteria**:
- POST /api/ask без body -- отправляет фиксированный текст нейросети

- IP хешируется (SHA256) и сохраняется для rate limiting
- Ошибка нейросети возвращает 503 с понятным сообщением
- Валидация: не более 10 запросов в минуту с одного IP (базовая)

**Файлы**:

#### [ ] 1.9 Real-time: SSE (MUST, 4ч, день 5)

**Описание**: Заменить polling на Server-Sent Events для мгновенного обновления ленты. SSE проще WebSocket и достаточен для однонаправленного потока.

**Acceptance Criteria**:
- GET /api/stream -- SSE эндпоинт, отправляет новые сообщения всем подключенным клиентам
- Клиент подключается к SSE при загрузке страницы
- Новые сообщения появляются у всех подключенных клиентов в течение 1 секунды
- Автореконнект при обрыве соединения (EventSource делает это нативно)
- Fallback на polling если SSE не поддерживается
- При подключении SSE -- polling отключается

## Риски и митигации

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| ЮKassa долго верифицирует магазин | Высокая | Подать заявку в день 1. Fallback: ручной перевод + подтверждение через Telegram-бот |
| OpenAI API блокирует запросы с контентом 18+ | Средняя | Системный промпт корректно формулирует задачу. Fallback на mock-ответы |
| Вирусный трафик > 10k rps | Низкая | SQLite выдержит чтение. Кешировать GET /messages. SSE масштабировать позже |
| Яндекс пришлет претензию за бренд "Алиса" | Низкая | Это пародия/мем. Можно переименовать в "Алёна" за 30 минут |
| Пользователи обходят ограничение 1 попытки | Средняя | Для MVP допустимо. Fingerprint + IP достаточно для 95% пользователей |

## Технические решения (зафиксированные)

| Вопрос | Решение | Обоснование |
|--------|---------|-------------|
| БД для MVP | SQLite (через Prisma) | Нулевая настройка, достаточно для старта. Миграция на PostgreSQL при необходимости |
| Real-time | SSE (Server-Sent Events) | Проще WebSocket, нативная поддержка браузеров, достаточно для однонаправленного потока |
| AI-провайдер | OpenAI GPT-3.5-turbo + mock fallback | Дешевый, быстрый, хорошо понимает русский. Mock как страховка |
| Оплата | ЮKassa | Самая простая интеграция для РФ-юрлица/ИП |
| Идентификация | FingerprintJS + IP hash | Не идеально, но достаточно для MVP. Не требует регистрации |
| Деплой | VPS + PM2 + nginx | Простой, контролируемый, дешевый |
| CSS | Tailwind CSS | Быстрая верстка, встроен в Next.js |

## Definition of Done (для каждой задачи)

1. Код написан и не ломает существующий функционал
2. Нет console.error в браузере
3. Работает на мобильном (Chrome DevTools эмуляция)
4. Переменные окружения вынесены в .env (нет хардкода ключей)
5. Базовый error handling (try/catch, toast при ошибке)

## MVP Scope -- что НЕ входит

Для ясности -- следующие фичи НЕ нужны для MVP (можно добавить позже):

- Регистрация / авторизация пользователей
- Админ-панель (достаточно прямых API-вызовов)
- Мультиязычность
- PWA / push-уведомления
- Генерация изображений (только текстовые ответы)
- Кеширование через Redis
- CDN для статики
- CI/CD pipeline (ручной деплой через скрипт)
- Тесты (unit/integration) -- пишутся после MVP если проект взлетит
