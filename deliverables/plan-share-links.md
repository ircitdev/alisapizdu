# План доработки: Персонализированные ссылки-приглашения

## Концепция

Пользователь нажимает "Поделиться" → заполняет имя друга → получает короткую ссылку.
Друг переходит по ссылке → нажимает "Спросить Алису" → сообщение публикуется от указанного имени.
Если "Разрешить менять" выключено — имя нельзя изменить, оно зафиксировано отправителем.

**Вирусный потенциал:** "Я отправил ссылку Васе, и теперь в ленте: `Вася — Алиса покажи пизду`"

---

## Этап 1: Backend — Модель данных

### 1.1. Новая таблица `invite_links`

```sql
CREATE TABLE invite_links (
  id TEXT PRIMARY KEY,           -- короткий код (6-8 символов, nanoid)
  created_by_ip TEXT,            -- хеш IP создателя
  created_by_user_id INTEGER,    -- ID создателя
  preset_name TEXT NOT NULL,     -- имя которое задал создатель
  allow_rename BOOLEAN DEFAULT 0,-- разрешить менять имя
  notify_email TEXT,             -- email для уведомлений (NULL = без уведомлений)
  message_id INTEGER,            -- ID сообщения (заполняется после использования)
  used_at TEXT,                  -- когда использована (NULL = не использована)
  used_by_ip TEXT,               -- хеш IP того кто использовал
  created_at TEXT NOT NULL
);
```

### 1.2. API endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/invite` | Создать ссылку-приглашение |
| GET | `/api/invite/:code` | Получить информацию о ссылке |
| POST | `/api/invite/:code/use` | Использовать ссылку (отправить сообщение) |

#### POST /api/invite

Запрос:
```json
{
  "preset_name": "Вася",
  "allow_rename": false,
  "notify_email": "user@example.com"
}
```

Ответ:
```json
{
  "code": "xK9mQ2",
  "url": "https://алисапокажипизду.рф/i/xK9mQ2",
  "preset_name": "Вася",
  "allow_rename": false
}
```

#### GET /api/invite/:code

Ответ:
```json
{
  "code": "xK9mQ2",
  "preset_name": "Вася",
  "allow_rename": false,
  "used": false,
  "created_by_user_id": 1000021
}
```

#### POST /api/invite/:code/use

Запрос (если allow_rename = true):
```json
{
  "name": "Василий Петрович",
  "timezone": "Europe/Moscow"
}
```

Ответ: аналогично POST /api/ask

## Этап 2: Backend — Логика

### 2.1. Создание ссылки (`POST /api/invite`)
- Проверить что пользователь уже отправлял сообщение (есть в rate limit map)
- Генерировать код через nanoid (6 символов)
- Сохранить в таблицу `invite_links`
- Лимит: максимум 10 ссылок на пользователя

### 2.2. Использование ссылки (`POST /api/invite/:code/use`)
- Проверить что ссылка существует и не использована (`used_at IS NULL`)
- Проверить что пользователь сам не создавал эту ссылку (по IP)
- Если `allow_rename = false` — игнорировать переданное имя, использовать `preset_name`
- Если `allow_rename = true` — использовать переданное имя (или `preset_name` по умолчанию)
- Создать сообщение с `type = 'invite'` (новый тип)
- Пометить ссылку как использованную
- Запустить генерацию ответа Алисы
- Rate limit НЕ применяется (ссылка = одноразовый пропуск)
- Если указан `notify_email` — отправить уведомление после генерации ответа Алисы

### 2.3. Email-уведомления

Отправка через SMTP (Яндекс Почта / любой SMTP-сервис).

**Когда отправлять:** после того как Алиса ответила на сообщение по ссылке-приглашению.

**Тема письма:**
`Ваша ссылка сработала! {preset_name} спросил Алису`

**Тело письма (HTML):**
```
Привет!

Кто-то перешёл по вашей ссылке и спросил Алису.

Имя: {имя пользователя}
Устройство: {device}, {os}
Город: {city}, {country}
Время: {datetime}

Сообщение: {user_message}

Ответ Алисы: {alice_response}

Посмотреть в ленте:
https://алисапокажипизду.рф/#msg-{message_id}

---
алисапокажипизду.рф
Экспериментальная платформа изучения поведения нейросетей
```

**Конфигурация (.env):**
```
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USER=notify@yourdomain.ru
SMTP_PASS=app-password
SMTP_FROM=Алиса покажи пизду <notify@yourdomain.ru>
```

**Ограничения:**
- Валидация email на клиенте и сервере (regex + длина)
- Не хранить email в открытом виде — хешировать для логов
- Максимум 10 уведомлений на один email в сутки (защита от спама)
- Email не показывается публично нигде, только в записи invite_links

### 2.4. Генерация сообщения
- Текст сообщения генерируется как обычно (рандомный вариант с "Алиса покажи пизду")
- В контексте для AI добавить: `[Приглашён пользователем #1000021]`
- Сообщение помечается `type = 'invite'` для отличия от free/paid

## Этап 4: Frontend — Страница приглашения

### 4.1. Роут `/i/[code]`

Новая страница `app/i/[code]/page.tsx`:

- Загружает данные ссылки через `GET /api/invite/:code`
- Если ссылка использована → показать "Ссылка уже использована"
- Если активна → показать:
  - "Вас пригласил пользователь #1000021"
  - "Ваше имя: Вася" (если allow_rename — поле редактируемое)
  - Кнопка "Спросить Алису" (на всю ширину)
  - Под кнопкой: текст "Сообщение будет опубликовано от имени Вася"

### 4.2. После отправки
- Редирект на главную страницу с якорем к опубликованному сообщению
- Или показать ответ Алисы прямо на странице приглашения

## Вирусная механика

```
Пользователь A нажимает "Спросить Алису"
  → Видит ответ
  → Нажимает "Поделиться"
  → Вписывает "Петя" → получает ссылку
  → Отправляет Пете в Telegram

Петя переходит по ссылке
  → Видит "Вас пригласил #1000021. Ваше имя: Петя"
  → Нажимает "Спросить Алису"
  → В ленте: "Петя — Алиса покажи пизду" + ответ
  → Петя видит ответ и хочет пригласить кого-то ещё
  → Цикл повторяется
```

Каждый пользователь становится распространителем — это реферальная петля без денежной мотивации, чисто на юморе.
