# Multi-Agent Workflow: "Алиса покажи пизду"

> Юмористический вирусный веб-проект. Адаптация схемы [workflow-with-memory](https://github.com/msitarzewski/agency-agents/blob/main/examples/workflow-with-memory.md) под конкретную идею.

## Концепция

Одностраничник с публичным чатом "с Алисой". Кнопка отправляет провокационный запрос нейросети. Платные кастомные сообщения за 1000р (1 попытка). Вся переписка видна всем в реальном времени.

## Команда агентов

| Агент | Роль |
|-------|------|
| Sprint Prioritizer | Разбивка на 2-недельный план (MVP за 2 недели) |
| UX Researcher | Анализ вирусных проектов-аналогов |
| Backend Architect | API, база, интеграция с нейросетью, оплата |
| Frontend Developer | Чат-интерфейс, real-time обновления |
| Rapid Prototyper | Быстрый запуск первой версии |
| Growth Hacker | Вирусный запуск (мемы, соцсети) |
| Reality Checker | Проверка на каждом этапе |

### Шаг 3 — Backend Architect

```
Activate Backend Architect.

Проект: "Алиса покажи пизду".
Recall sprint plan и research brief.

Стек: Node.js, PostgreSQL (или SQLite для MVP), WebSocket.

Спроектируй:
1. Схема БД:
   - messages (id, type[system|paid], sender_name, text, alice_response, created_at)
   - payments (id, message_id, amount, status, provider_id)
2. API:
   - POST /ask — бесплатный запрос (rate limit по IP)
   - POST /ask-custom — платный запрос (после оплаты)
   - GET /messages — лента всех сообщений (пагинация + SSE/WebSocket)
   - POST /payment/create — создание платежа
   - POST /payment/callback — вебхук от платёжки
3. Интеграция с нейросетью:
   - Какой API использовать для "Алисы" (YandexGPT / YandexART / заглушка)
   - Стратегия модерации ответов
4. Rate limiting и защита от абуза

Remember deliverables tagged: alisapizdu, api-spec, schema, frontend-developer.
```

## Неделя 2: Сборка + Запуск

### Шаг 4 — Frontend Developer

```
Activate Frontend Developer.

Проект: "Алиса покажи пизду".
Recall API spec и research brief.

Построй интерфейс:
- Стек: Next.js, TypeScript, Tailwind CSS
- Одна страница, без навигации
- Вверху: заголовок + счётчик сообщений
- Центр: лента чата (скролл, автообновление через WebSocket)
  - Сообщения пользователей (имя/аноним + текст)
  - Ответы "Алисы" (с аватаркой, стилизация под Яндекс)
- Внизу: кнопка "Спросить Алису" (большая, яркая)
  - После нажатия: анимация ожидания, ответ появляется в ленте
  - Кнопка "Написать своё (1000р)" → модальное окно с полем ввода и оплатой
- Кнопка "Поделиться" на каждом сообщении (копирует скриншот/ссылку)
- Тёмная тема, стиль "Яндекс Алиса meets мем"

Дизайн должен быть мемным и скриншотабельным.
Remember progress tagged: alisapizdu, frontend.
```
